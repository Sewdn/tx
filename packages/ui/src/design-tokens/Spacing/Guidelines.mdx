## The token system

<p className="text-lg text-muted-foreground leading-[1.6] m-0 max-w-[680px]">
  Five named tokens control every gap. Each names the containing scope: from a single phrase out to
  the page frame. Tune once in CSS; every product surface updates together.
</p>

<div className="rounded-xl border-2 border-primary/20 bg-muted/40 p-section mt-section">
  <h4 className="text-base font-bold text-foreground m-0 mb-component">Why tokens, not pixels</h4>
  <p className="text-base text-muted-foreground leading-[1.6] m-0">
    A token like <strong>section</strong> resolves to different pixel values by density and
    breakpoint. Change it once and every card, form, and layout updates — no component edits.
  </p>
</div>

<p className="text-base text-muted-foreground leading-[1.6] m-0 max-w-[680px] mt-section">
  Ask <em>where does this spacing live?</em> Icon beside a label → <strong>micro</strong>. Button
  padding → <strong>component</strong>. Card interior → <strong>section</strong>. Between page
  blocks → <strong>region</strong>. Page shell margins → <strong>boundary</strong>.
</p>
